# Phase 0 — Bloquant production

**Statut global : 🟡 Planifié — non commencé**
**Horizon estimé : 3–5 semaines**
**Objectif :** rendre une ouverture de production défendable. Sans ces correctifs : fraude financière possible, fuite de données inter-tenant, flux métier silencieusement cassés, corruption de données au premier scale-out.

> **Mode d'emploi des cases à cocher.** La source de vérité est ce fichier Markdown, versionné dans git : cochez `- [ ]` → `- [x]` dans l'IDE (ou tout éditeur) et committez, au fur et à mesure de l'implémentation, comme n'importe quel changement de code. La page HTML régénérée (`python3 docs/audits/build_reports.py`) reflète l'état du fichier au moment du build et affiche une barre de progression calculée automatiquement. **Les cases sont en lecture seule dans le navigateur** — on ne coche jamais depuis la page HTML, uniquement en éditant ce fichier.

Chaque item référence son audit source et sa localisation exacte. Aucun constat des 7 audits n'est omis — les items non retenus pour la Phase 0 sont dans [Phase 1](phase-1-hardening.html) ou [Phase 2](phase-2-target-architecture.html), jamais silencieusement abandonnés.
